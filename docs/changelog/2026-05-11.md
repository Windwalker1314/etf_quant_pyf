# 2026-05-11

## 本次变更

- 搭建了日频 ETF 量化框架，支持数据下载、因子加工、策略、回测、walk-forward、每日调仓计划。
- 迁移了 BigQuant 的 ETF 轮动策略到本项目，包含：
  - 多因子打分
  - 动态 score gap
  - 止损 / 移动止损 / 止盈
  - 冷却期
  - 逆波动配仓
  - 簇权重上限
- 接入了三类数据源：
  - `yahoo`
  - `akshare`
  - `tushare`
- 新增了数据质量检查命令 `validate-data`。
- 新增了 conda 交互式环境建议，避免污染系统 Python。

## 关键配置

- `configs/bigquant_rotation_yahoo.yaml`
- `configs/bigquant_rotation_akshare.yaml`
- `configs/bigquant_rotation_tushare.yaml`
- `configs/bigquant_rotation_hybrid.yaml`

## 实验结果

### 1. Yahoo 调整价回测

命令：

```bash
PYTHONPATH=src .conda/etf-quant/bin/python -m etf_quant.cli backtest --config configs/bigquant_rotation_yahoo.yaml
```

结果：

- 年化收益：`26.33%`
- 年化波动：`17.10%`
- 夏普：`1.45`
- 最大回撤：`-11.54%`
- 总收益：`292.71%`

结论：

- 这是目前最接近原策略表现的口径之一。
- Yahoo 当前在本机网络下可下载历史缓存，但现网重拉时会遇到 `403 Forbidden`。

### 2. AkShare 回测

命令：

```bash
PYTHONPATH=src .conda/etf-quant/bin/python -m etf_quant.cli backtest --config configs/bigquant_rotation_akshare.yaml
```

结果：

- 年化收益：`25.74%`
- 年化波动：`22.23%`
- 夏普：`1.15`
- 最大回撤：`-37.09%`
- 总收益：`282.48%`

结论：

- 回撤偏大不是策略主逻辑问题，主要是 AkShare fallback 到 Sina 后的部分 ETF 数据出现未复权跳变。
- 已定位到 `513100.SH` 和 `513500.SH` 的异常大跌幅日。

### 3. Hybrid 回测

数据口径：

- Yahoo 调整价为主
- 只用 AkShare 补 `160723.SZ`

命令：

```bash
PYTHONPATH=src .conda/etf-quant/bin/python -m etf_quant.cli backtest --config configs/bigquant_rotation_hybrid.yaml
```

结果：

- 年化收益：`32.49%`
- 年化波动：`18.74%`
- 夏普：`1.60`
- 最大回撤：`-16.76%`
- 总收益：`419.57%`

结论：

- 这是当前最可信的项目内对照结果。
- 比 AkShare fallback 版稳定很多。

## 数据质量检查

命令：

```bash
PYTHONPATH=src .conda/etf-quant/bin/python -m etf_quant.cli validate-data --config configs/bigquant_rotation_akshare.yaml
PYTHONPATH=src .conda/etf-quant/bin/python -m etf_quant.cli validate-data --config configs/bigquant_rotation_hybrid.yaml
```

结果：

- AkShare 原始缓存发现两个异常跳变：
  - `513100.SH` on `2022-01-14`: `-80.45%`
  - `513500.SH` on `2022-03-30`: `-49.18%`
- Hybrid 仅剩一个 `159915.SZ` 的 `20%` 级别跳动，可接受为市场本身极端波动或涨停级别事件。

## 当前建议

1. 研究和回测优先使用 `bigquant_rotation_hybrid.yaml`。
2. 若要复现 BigQuant 更接近的口径，优先再接 Tushare 或 Wind。
3. 若继续用 AkShare，必须先做数据质量检查，避免未复权 fallback 数据污染结果。
