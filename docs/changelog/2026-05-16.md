## 本次目标

- 接着 `2026-05-15` 最新 changelog 的结论做下一轮优化。
- 不继续单点追 `max_position_weight`，优先验证“趋势状态预算”是否能改善 global 长样本，尤其是 `2013-2018` 弱段和 `2018 / 2022` 回撤。

逻辑：

- 用目标趋势资产相对防御资产的归一化强弱，以及快慢均线状态，划分 `strong / neutral / weak`。
- `strong` 时放宽目标 bucket 和权益组预算。
- `weak` 时收紧目标 bucket 和权益组预算。
- 只覆盖 bucket / group caps，不改变原有排名器、止盈止损、单资产上限和交易缓冲逻辑。

新增研究入口：

- `sweep-trend-budget`：扫描趋势状态预算参数，可选 `--walk-forward`。

新增配置：

- `configs/global_etf_trend_budget_yahoo.yaml`

## 分段观察

对比 global 修正后 position-cap walk-forward：

| 口径 | 区间 | 年化收益 | 夏普 | 最大回撤 |
| --- | --- | ---: | ---: | ---: |
| position-cap | 2012-02-06 到 2020-02-05 | 1.68% | 0.22 | -14.88% |
| trend-budget | 2012-02-06 到 2020-02-05 | 3.29% | 0.40 | -15.77% |
| position-cap | 2020-02-06 到 2026-05-11 | 18.94% | 1.25 | -16.91% |
| trend-budget | 2020-02-06 到 2026-05-11 | 16.18% | 1.17 | -20.07% |

结论：

- 趋势预算确实改善了 `2012-2020` 低收益段，但改善幅度不够大。
- 它牺牲了 `2020` 后样本外收益，并且没有解决最大回撤问题。
- 目前不能替代 `global_etf_position_cap_yahoo.yaml` / `bigquant_rotation_hybrid_position_cap.yaml` 主候选。
- 它可以保留为下一轮“连续风险预算”的基础设施。
